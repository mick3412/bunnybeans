## 進貨驗收：退回供應商（return-to-supplier）手測清單

### 前置條件
- 後端可連線（`VITE_API_BASE_URL` 正確）
- 具備可操作的驗收單（Receiving Note）且狀態為 `COMPLETED`

### 步驟
1. 進入 `/admin/receiving-notes`，打開任一 `COMPLETED` 驗收單詳情
2. 在「退回供應商」區塊：
   - 任選 1～2 列輸入退回數量（不得超過合格數）
   - 輸入原因（例如：瑕疵/規格不符/效期問題）
3. 點擊「送出退回供應商」
4. 觀察：
   - toast 顯示成功或失敗訊息
   - 成功後會顯示「本次送出明細」（列、數量、原因）
   - 詳情會刷新（後端會寫入庫存事件 `RETURN_TO_SUPPLIER` 與金流事件 `PURCHASE_RETURN`）
5. 故障情境（可用不合法數量或重複退回觸發後端校驗）：
   - 會顯示後端錯誤訊息（例如：超過可退數量）
