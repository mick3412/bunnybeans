# 前端 UI 修改原則與方向

> 本文件依現有 mockup（[docs/mockup/](mockup/)）的**儀表板／數據感**風格整理，供前端與 Agent 進行 UI 修改時遵循。**實際排版（尤其主畫面區）可依規格與需求重新調整**，但修改方向須符合本文件。

---

## 一、適用範圍

- **後台 Admin**：總覽、會員與集點、營運（金流報表等）、庫存／商品／採購等。
- **POS**：收銀、訂單、促銷、報表。
- 新頁面或既有頁面重構時，視覺與互動須與本原則一致；主內容區的欄位順序、區塊多寡、單欄／雙欄可依產品規格與需求決定。

---

## 二、必須遵守（原則）

以下為**不可偏離**的約定，修改 UI 時須維持。

### 2.1 語言與文案

- **介面以中文為主**：按鈕、標籤、表頭、導覽、提示、錯誤訊息、placeholder 一律繁體中文。
- **非必要不展示英文**：僅在技術必要時（專有名詞、品牌、API 代碼、traceId 等）使用英文。
- **全站無 emoji**：導覽、按鈕、狀態標籤僅用文字或圖示（若有），不用 emoji。

### 2.2 色系語意（儀表板／數據感）

| 用途 | 色碼／變數 | 說明 |
|------|------------|------|
| 主內容區背景 | `#f1f5f9` | 淺灰，與卡片白底區隔 |
| 側欄背景 | `#1e293b` | 深灰藍，固定 |
| 主行動（按鈕、連結、選中） | `#0ea5e9` | 主色；側欄選中可加左線 `#38bdf8` |
| 成功／完成狀態 | `#16a34a` | 綠；標籤、隔行可搭配淺綠底 |
| 警示／掛帳／折抵 | `#ea580c` 或 `#dc2626` | 橙或紅；小面積 |
| 中性／次要文字 | `#64748b` | 標籤、說明、KPI label |
| 卡片／表格邊框 | `#e2e8f0` | 極細邊框，不搶眼 |
| 主要文字 | `#1e293b` | 標題、內文 |

**規則**：大面積僅主背景與側欄深色；主色與狀態色用於焦點與狀態，不整版鋪底。

### 2.3 殼層結構

- **側欄**：深色底（#1e293b）、固定寬（約 12rem～14rem）。區分**大區塊**（區段標題，如「會員與集點」「收銀台」）與**小區塊**（該區下連結）；底部分隔線後為跨區操作（如「前往 POS」「後台」）。選中項：左側主色豎條（2～3px）或背景略深。
- **頂欄**：白底、細邊框；放置**單一頁級標題（H1）**、日期或篩選、必要時全域操作。
- **主內容區**：淺灰底；內邊距一致（約 1.5rem～2rem）；可設 max-width（如 1440px）置中。**主內容區內之版面（單欄／雙欄、區塊順序、是否 sticky 表頭等）由前端依規格與需求決定**，但須符合下述「方向」與元件語意。

### 2.4 字階與可讀性

- **字型**：`PingFang TC`, `Microsoft JhengHei`, `Noto Sans TC`, `system-ui`, `sans-serif`。
- **層級**：一頁單一 H1（頁標）；區塊標題 H2/H3；內文與表單 Body；輔助 Caption。**最小字級不低於 12px**（側欄、POS 窄欄同）。
- **行高**：內文 ≥ 1.5；表單／表格可 1.4～1.5。

## 三、修改方向（須遵循，細節可調）

以下為**建議的視覺與結構方向**；實際主畫面排版可依規格重排，但整體風格須與此一致。

### 3.1 數據感與 KPI

- **KPI 卡**：以**左側色條**區分類型（例如第 1 張藍、第 2 張綠、第 3 張橙、第 4 張灰），數值大字、標籤小字灰；白底、細邊框、圓角約 0.5rem。
- **摘要區塊**：列表頁或報表頁可於表上方放「摘要卡」或小統計（總筆數、狀態數等），與 mockup 的 `.mu-blocks`／`.mu-block-card` 概念一致。

### 3.2 表格

- **邊框**：1px、#e2e8f0，不粗、不搶眼。
- **表頭**：淺底（#f8fafc）、字重 600、與內容區分。
- **隔行**：可選極淺灰或淺綠隔行；hover 時背景略深以便掃描。
- **長列表**：建議支援**懸浮表頭**（sticky thead），捲動時表頭固定，高度可設上限（如 20rem）避免一頁過長。
- **連結**：表格內連結用主色 #0ea5e9，字重 500。

### 3.3 按鈕與表單

- **主要按鈕**：主色底、白字、圓角約 0.5rem；hover 略暗。
- **次要按鈕**：白底、灰邊、深色字；hover 淺灰底。
- **危險**：紅底小面積或白底紅邊／紅字。
- **表單**：輸入框白底、灰邊；**focus 時主色邊框 + 主色 ring**（如 2px 陰影）；錯誤／必填可用紅小字或紅邊。

### 3.4 區塊與版面

- **區塊標題**：小節標題與內容間有明確間距；可加底線或留白與下一個區塊區隔。
- **雙欄**：需並列時（如表單＋說明、圖表＋列表）可用雙欄；窄螢幕改單欄。
- **卡片**：白底、細邊框、圓角 0.5rem～0.75rem；區塊內 padding 一致。
- **底部操作列**：列表頁或報表頁可設**懸浮列**（sticky bottom）：顯示筆數、匯出、新增等；白底、頂邊線、與主內容 padding 對齊。

### 3.5 狀態標籤

- **成功／完成**：綠底（淺綠 rgba）＋深綠字。
- **進行中／資訊**：主色淺底＋主色系字。
- **警告／掛帳／折抵**：橙或紅淺底＋深橙／深紅字。
- 僅文字與細底，**無 emoji**。

---

## 四、主畫面區的彈性

- **區塊順序**：可依規格調動（例如先 KPI 再表、或先篩選再摘要再表）。
- **單欄／雙欄／多欄**：由需求決定；維持間距與層級一致即可。
- **是否 sticky 表頭**：依列表長度與使用情境決定。
- **是否懸浮底列**：依是否需要常駐「匯出／新增／筆數」等決定。
- **KPI 張數與左色條**：可 2～6 張；左色條建議維持藍／綠／橙／灰等區分，避免全同色。

## 六、總結

- **原則**：中文為主、無 emoji、色系語意固定、殼層結構固定、字階與可讀性下限固定。
- **方向**：數據感 KPI、細邊框表格、懸浮表頭與底列、主色按鈕與 focus、區塊標題與雙欄／卡片運用。
- **彈性**：主內容區之版面、區塊順序、單雙欄、是否 sticky／懸浮列，由前端依規格與需求調整，惟須符合上述原則與方向。
