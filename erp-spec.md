## Local Host ERP 系統規格說明

本文件描述一套部署於 Local Host 的 ERP 系統之完整規格，涵蓋：

- 商家 / 組織管理
- 供應商管理
- 商品與庫存管理
- 消費者 / 會員（CRM 基礎）
- 前端消費流程（POS 流程）
- 採購與庫存作業流程
- CRM 進階與行銷管理
- 財務與報表（與會計介面）
- 系統管理與安全
- 技術與前端大致規格

後續可依此文件進一步細化成資料庫欄位設計、API 規格與畫面 Wireframe。

---

## 一、商家 / 組織主檔管理

### 1.1 公司 / 商家基本資料

**目的**：維護單一或多家商戶之基本營運資訊，作為全系統預設參數來源。

- **資料欄位**
  - 基本資訊
    - 公司代碼
    - 公司名稱
    - 品牌名稱
    - 統一編號
    - 發票抬頭
    - 客服電話
    - 客服 Email
    - 公司地址
  - 營運資訊
    - 營業時間（平日 / 假日）
    - 國家 / 幣別（主幣別）
    - 預設稅率（一般稅率、零稅率）
    - 發票類型設定（電子發票 / 紙本發票 / 收據）
  - 財務相關
    - 預設會計科目對應碼（銷貨收入、進貨成本、存貨科目、應收、應付等）
    - 發票字軌 / 單號規則
  - 品牌視覺
    - 公司 Logo
    - 品牌顏色設定（供前端與列印樣式使用）

- **功能需求**
  - 新增 / 編輯 / 停用公司資料
  - 多品牌管理（同一公司底下多品牌）
  - 稅率與幣別設定（系統主幣別 + 參考匯率）

### 1.2 組織與門市 / 倉庫架構

**目的**：管理公司內部組織結構與實體門市 / 倉庫資訊，供權限與報表分層使用。

- **資料欄位**
  - 部門主檔
    - 部門代碼
    - 部門名稱
    - 上層部門
    - 負責人
  - 門市主檔
    - 門市代碼
    - 門市名稱
    - 門市地址
    - 聯絡電話
    - 營業時間
    - 所屬區域 / 區經理
    - 門市負責人
    - 預設銷售倉庫
    - 門市營運參數（是否允許退貨、是否允許負庫存、收款方式限制等）
  - 倉庫主檔
    - 倉庫代碼
    - 倉庫名稱
    - 倉庫類型（總倉 / 門市倉 / 虛擬倉）
    - 倉庫地址
    - 負責人

- **功能需求**
  - 建立 / 維護組織樹狀結構（總公司 > 區域 > 門市）
  - 門市與倉庫關聯設定（門市預設銷售倉庫）
  - 門市營業參數維護（退貨策略、庫存策略、收款方式開關）

## 二、供應商管理模組

### 2.1 供應商主檔

**目的**：建立供應商基本資料與採購條件，作為採購與應付帳款之基礎。

- **資料欄位**
  - 基本資料
    - 供應商代碼
    - 供應商名稱
    - 統一編號
    - 聯絡人姓名
    - 聯絡電話
    - Email
    - 地址
  - 財務資料
    - 付款條件（現金 / 30天 / 60天 等）
    - 信用額度
    - 預設幣別
    - 匯款帳號（銀行名稱、分行、帳號）
  - 採購設定
    - 預設交貨地址
    - 預設交貨倉庫
    - 交貨天數（Lead time）
    - 最小訂購量
    - 折扣條件（長期合約折扣）
  - 品質 / 評等
    - 供應商等級
    - 供應商評分
    - 黑名單標記（是否為風險供應商）

- **功能需求**
  - 新增 / 編輯 / 停用供應商
  - 供應商品項關聯（同一商品可對應多供應商，不同進價與條件）
  - 供應商文件附件管理（合約、價目表 PDF 等）

### 2.2 供應商往來記錄

**目的**：掌握供應商歷史交易與績效，輔助採購決策。

- **功能需求**
  - 採購訂單列表（依供應商篩選）
  - 進貨記錄與退回供應商記錄查詢
  - 付款記錄與應付帳款餘額查詢
  - 供應商績效報表
    - 準時交貨率
    - 退貨率
    - 平均進價變動

## 三、商品與庫存主檔管理

### 3.1 商品主檔

**目的**：集中管理所有商品資訊，供銷售、採購、庫存與行銷模組共用。

- **基本結構**
  - 商品代碼（SKU）
  - 條碼（支援多條碼）
  - 商品名稱（可多語系）
  - 商品分類（大類 / 中類 / 小類，多層級）
  - 品牌
  - 系列 / 型號
  - 規格（如顏色、尺寸等，可作為子品項屬性）
  - 單位
    - 主單位（如：件）
    - 輔助單位（如：箱、打）
    - 單位換算率

- **價格與成本**
  - 建議售價
  - 一般門市售價（可依門市或通路不同價）
  - 會員不同價層（一般會員 / VIP 等）
  - 進貨成本
    - 最近一次進價
    - 平均成本
    - 標準成本
  - 促銷價設定
    - 促銷售價
    - 起訖日期
    - 適用門市 / 通路
    - 適用會員群組 / 等級

- **屬性與標籤**
  - 是否序號商品（需逐一序號管理，如 3C 產品）
  - 是否批號 / 有效期管理（食品、藥品）
  - 是否折扣排除（不可參與折扣）
  - 是否點數累積排除（不累積點數）
  - 自訂標籤（熱銷、新品、清倉、自有品牌等）

- **多媒體與說明**
  - 商品主圖片
  - 附加圖片（多張）
  - 商品短描述（列表顯示用）
  - 商品詳細描述（詳細頁顯示用）

- **功能需求**
  - 新增 / 編輯 / 停用商品
  - 大量匯入 / 匯出（CSV / Excel）
  - 價格批次調整（依類別、供應商、百分比調整等）

### 3.2 庫存主檔與倉位管理

**目的**：對商品在各倉庫 / 門市的庫存量進行精準控管。

- **庫存層次**
  - 公司
  - 倉庫
  - 商品
  - 批號（選用）
  - 序號（選用）

- **庫存屬性欄位**
  - 安全庫存量
  - 最高庫存量
  - 補貨點
  - 是否允許負庫存（可分商品 / 倉庫層級）

- **倉位（儲位）管理（選配）**
  - 倉位代碼
  - 倉位名稱 / 區域
  - 倉位容量
  - 商品與倉位關聯（預設儲位 / 多儲位）

- **功能需求**
  - 查詢即時庫存（依商品 / 倉庫 / 門市）
  - 批量設定安全庫存與補貨點

## 四、消費者 / 會員管理（CRM 基礎）

### 4.1 消費者 / 會員主檔

**目的**：集中管理消費者與會員資料，供 POS、行銷與報表使用。

- **資料欄位**
  - 基本資訊
    - 會員編號
    - 姓名 / 暱稱
    - 性別
    - 生日
    - 手機號碼
    - Email
    - 地址
  - 帳戶資訊
    - 會員卡號（如有實體卡）
    - 會員等級
    - 註冊來源（門市 / 線上 / 匯入）
    - 註冊日期
  - 行為 / 分析
    - 偏好品類
    - 平均客單價
    - 購買頻率
    - 最近一次消費日期
  - 同意與合規
    - 簡訊行銷同意狀態
    - Email 行銷同意狀態
    - 個資同意紀錄（時間戳與版本）

- **功能需求**
  - 新增 / 編輯 / 停用會員
  - 會員合併（重複帳號合併）
  - 會員標籤管理（自訂標籤、多選）
  - 黑名單管理（異常退貨或違規客戶）

### 4.2 會員資產管理（點數 / 優惠券 / 餘額）

**目的**：管理會員所有可用「資產」，以支援行銷與回饋機制。

- **點數**
  - 規則設定
    - 點數累積比例（例如：每消費 1 元累積 1 點）
    - 不同會員等級倍數點數（VIP 1.2 倍）
    - 指定商品或活動多倍點
    - 點數有效期（固定期限或滾動期限）
  - 資料結構
    - 會員點數餘額
    - 點數交易明細（累積 / 抵扣 / 調整）
  - 功能
    - POS 結帳時顯示可使用點數與預計累積點數
    - 點數手動調整（需權限，並記錄原因）

- **優惠券 / 折價券**
  - 券種
    - 金額折抵券（滿額折抵固定金額）
    - 折扣券（例如 9 折）
    - 贈品券（買指定商品送贈品）
    - 運費券（如有線上通路）
  - 規則與限制
    - 使用門檻（滿額 / 滿件）
    - 限定商品 / 類別
    - 限定門市 / 通路
    - 限定會員等級 / 分群
    - 使用期間
    - 每人限用次數
  - 功能
    - 優惠券發放（單筆、批次、活動自動發）
    - 優惠券核銷（POS / 線上）
    - 使用紀錄與剩餘次數查詢

- **儲值 / 餘額（選配）**
  - 會員儲值帳戶
  - 儲值交易（儲值、贈送金額）
  - 消費扣抵紀錄
  - 到期管理（若有儲值金有效期）

## 五、前端消費流程（POS / 結帳流程）

### 5.1 銷售流程（單一交易）

**目標**：提供收銀員迅速、穩定且錯誤率低的結帳流程。

**流程步驟：**

1. **開啟銷售單**
   - 選擇門市 / 收銀機台（可由登入帳號自動帶出）
   - 自動帶出收銀員資訊與當班編號
2. **選擇客戶**
   - 掃描會員條碼
   - 以手機號 / 姓名 / 卡號查詢會員
   - 若無會員，可選擇匿名消費
3. **加入商品**
   - 掃描商品條碼
   - 手動輸入 SKU 或關鍵字搜尋商品
   - 可調整數量、單價（依權限）、單品折扣
   - 系統自動套用有效促銷（滿額折、滿件折、第二件半價、加價購等）
4. **檢視促銷與點數**
   - 顯示本筆可用的優惠券清單，供收銀員選擇使用
   - 顯示可使用點數與預計累積點數
5. **選擇支付方式**
   - 支援多種支付方式：
     - 現金
     - 信用卡 / 金融卡
     - 電子支付（Line Pay、街口等，可自訂）
     - 儲值金（如有啟用）
   - 支援多支付混合支付（如：部分現金 + 部分信用卡）
6. **確認並結帳**
   - 顯示應收金額、實收金額、找零金額
   - 結帳後產生銷售憑證（發票 / 收據）
   - 小票列印與電子憑證（Email / 簡訊）
   - 產生銷售主檔與明細、更新庫存，更新會員點數與優惠券
7. **資料同步與稽核**
   - 本機與伺服器資料同步（如為分散架構）
   - 寫入操作 Log（如手動改價、手動折扣）

### 5.2 退貨與換貨流程

**目的**：提供可控且可稽核的售後流程，避免濫用與損失。

- **退貨（有小票 / 發票）**
  1. 由銷售單號 / 發票號搜尋原交易
  2. 選擇要退貨的商品與數量
  3. 檢查退貨期限、會員身分（是否需同一會員 / 證件）
  4. 設定退貨原因（品質問題、尺寸不合、操作錯誤等）
  5. 執行退貨
     - 依原支付方式退款（或依公司政策改為現金 / 代金券）
     - 庫存回倉：可選「可銷售庫存」或「瑕疵倉」
  6. 更新應收 / 應付與會員點數（退回累積點數、回收活動贈點等）

- **無票退貨 / 換貨**
  - 依系統參數決定是否允許
  - 可設定：
    - 單筆最高退貨金額
    - 每會員 / 每日退貨次數限制
    - 必須管理員授權
  - 換貨流程（舊品退貨 + 新品銷售）
    - 退貨舊品（生成退貨單）
    - 建立新品銷售單
    - 差額補收或退款

### 5.3 班表與收銀結帳

**目的**：控管收銀員現金與交易狀態，支援對帳與稽核。

- **開班**
  - 收銀員登入系統
  - 設定起始現金金額
  - 系統開立「當班紀錄」

- **班中管理**
  - 查詢當班銷售金額、交易筆數、退貨筆數
  - 監控現金抽屜金額（可選）

- **結班 / 日結**
  - 系統統計各支付方式的應收金額
  - 收銀員輸入實際點交金額，系統計算差異
  - 產生結帳報表（每日營業報表、支付方式統計）
  - 結束後鎖定該班別交易（修改需特別權限）

## 六、採購與庫存作業流程

### 6.1 採購計畫與採購單

**目的**：根據庫存與銷售狀況，進行系統化的補貨與採購。

- **採購建議**
  - 依商品「安全庫存 / 補貨點 / 最高庫存」自動產生補貨建議
  - 可參考指定期間銷售歷史（例如最近 30 天）
  - 可依供應商 / 類別彙總建議

- **採購單建立**
  - 依供應商建立採購單
  - 採購單頭
    - 供應商
    - 預計到貨日期
    - 交貨倉庫
    - 付款條件
    - 幣別與匯率（如有）
  - 採購單明細
    - 商品
    - 數量
    - 單價
    - 折扣（行折扣 / 整筆折扣）
  - 審批流程
    - 小額自動核准
    - 大額需主管線上審批

### 6.2 驗收 / 進貨流程

**目的**：將供應商進貨轉為正式庫存與應付帳款。

- **流程**
  1. 進貨人員選擇「進貨驗收」
  2. 由採購單產生待驗收單（或直接建立無訂單進貨）
  3. 核對供應商送貨數量、批號 / 有效期、單價異動
  4. 輸入實收數量與差異原因（短少、破損、多送）
  5. 驗收完成後：
     - 更新指定倉庫庫存
     - 產生進貨單 / 進貨明細
     - 產生應付帳款（AP）

### 6.3 退回供應商（退貨）

**目的**：處理不良品、滯銷品等退回供應商。

- **流程**
  - 建立「退供單」
    - 可由歷史進貨單選取商品與批號
  - 輸入退貨數量與退貨原因
  - 確認後：
    - 扣減庫存
    - 產生應付沖銷或折讓記錄

### 6.4 調撥流程（倉庫間 / 門市間）

**目的**：在倉庫與門市之間移動商品，以平衡庫存。

- **流程**
  - 調撥申請
    - 門市提出補貨申請（向總倉）
  - 調出作業
    - 調出倉庫依申請單出貨，生成調撥單，扣減調出倉庫庫存
  - 調入作業
    - 調入倉庫 / 門市進貨驗收，確認數量後增加庫存

### 6.5 盤點流程

**目的**：確保帳面與實際庫存一致，並管理損失。

- **盤點計畫**
  - 全倉盤點 / 抽盤特定品類 / 高價商品盤點
  - 設定盤點日期、負責人
  - 盤點期間是否凍結相關庫存異動（可選）

- **盤點執行**
  - 產生盤點表或使用手持裝置掃碼盤點
  - 輸入實盤數量

- **盤盈 / 盤虧處理**
  - 系統計算差異數量與金額
  - 產生盤盈 / 盤虧單
  - 由主管審批後正式調整庫存

## 七、CRM 進階功能與行銷管理

### 7.1 會員分級與分群

**目的**：透過分級與分群，有效執行差異化行銷策略。

- **會員分級**
  - 分級規則
    - 依累積消費金額
    - 依交易次數
    - 依近 N 個月活躍度
  - 自動升降級機制（排程批次處理）
  - 手動調整等級（需權限，紀錄原因）

- **會員分群**
  - 動態分群
    - 依條件即時計算是否屬於某群組
  - 靜態分群
    - 特定時間點匯出快照，鎖定名單
  - 常見條件
    - 年齡區間
    - 性別
    - 偏好品類
    - 平均客單價
    - 最近一次消費時間
    - 消費頻率

### 7.2 行銷活動設定

**目的**：集中管理所有促銷活動規則，供 POS 自動套用。

- **活動類型**
  - 價格促銷
    - 滿額折
    - 滿件折
    - 第二件 / 第三件優惠
    - 買 A 送 B（贈品）
  - 點數活動
    - 全館多倍點
    - 指定品類多倍點
  - 優惠券活動
    - 註冊送券
    - 生日送券
    - 回購券（在指定期間內未回購者發券）

- **適用範圍**
  - 指定門市 / 通路
  - 指定商品 / 類別
  - 指定會員等級 / 分群
  - 活動期間與每日時段限制

- **執行與追蹤**
  - POS 銷售單自動判斷套用活動
  - 活動效果報表
    - 參與交易筆數
    - 銷售額與毛利
    - 活動成本與 ROI 粗估

### 7.3 客戶互動紀錄（簡易 CRM）

**目的**：記錄與客戶的溝通與服務歷程。

- **功能需求**
  - 建立客服聯絡紀錄（來電 / 投訴 / 意見 / 咨詢）
  - 為會員新增備註與追蹤事項
  - 匯出名單（供外部簡訊 / EDM 平台使用）

## 八、財務與報表（與會計介面）

> 若本系統不直接做總帳，可先聚焦於應收 / 應付與對帳，並提供匯出檔案給會計系統。

### 8.1 應收帳款（AR）

- 來源
  - 銷售單（產生應收）
  - 退貨單（產生折讓）
- 功能
  - 客戶掛帳（B2B 客戶）設定帳期
  - 應收明細與餘額查詢
  - 應收帳齡分析（0–30、31–60、61–90、90 以上）
  - 逾期提醒列表

### 8.2 應付帳款（AP）

- 來源
  - 進貨單（產生應付）
  - 退供單（產生折讓或沖銷）
- 功能
  - 供應商應付明細與餘額查詢
  - 帳齡分析
  - 付款計畫與付款建議（依到期日、金額排序）

### 8.3 收付款作業

- 收款
  - 對應應收帳款（抬頭、金額、日期）
  - 支援多種收款方式（現金、轉帳等）
- 付款
  - 對應應付帳款
  - 支援多種付款方式
- 對帳
  - 可匯出對帳資料供銀行對帳

### 8.4 報表中心

- **營運報表**
  - 每日 / 每月銷售報表（按門市、商品、類別、品牌）
  - 毛利分析報表
  - 門市績效排行
  - 銷售人員績效

- **庫存報表**
  - 即時庫存表
  - 庫存週轉率報表
  - 呆滯品清單（長期無銷售）
  - 到期品報表（有批號 / 有效期管理時）

- **會員 / CRM 報表**
  - 會員數成長趨勢
  - 會員活躍度分析
  - 會員等級分布
  - 行銷活動成效分析

- **財務對帳報表**
  - 銷售與收款對帳表
  - 進貨與付款對帳表
  - POS 支付方式彙總與銀行入帳核對表

## 九、系統管理與安全

### 9.1 帳號與權限

- **使用者帳號**
  - 帳號、姓名
  - 所屬部門 / 門市
  - 職稱

- **角色與權限模型**
  - 角色（如：系統管理員、門市主管、收銀員、採購、倉管等）
  - 功能權限（檢視 / 新增 / 編輯 / 刪除 / 審批）
  - 資料範圍（僅能看自己門市、所屬區域、或全部門市）

- **登入安全**
  - 密碼規則（長度、複雜度）
  - 密碼有效期與定期更換
  - 多次登入失敗鎖定機制

- **操作日誌（Audit Log）**
  - 紀錄重要操作：
    - 價格變更
    - 庫存手動調整
    - 手動點數 / 儲值金調整
    - 審批通過與駁回紀錄

### 9.2 系統參數設定

- 公司 / 門市層級參數
  - 是否允許負庫存
  - 是否允許無票退貨 / 換貨
  - 會員點數換算比例
  - 發票 / 單據號碼規則

- 前端顯示與列印格式
  - 小票樣式
  - Logo 與頁尾文字（退貨規則、客服資訊）

- 其他
  - 時區設定
  - 日期與數字格式

### 9.3 備份與維運（Local Host 重點）

- 備份機制
  - 排程備份（每日 / 每週）
  - 備份目標路徑（本機 / 網路磁碟）
  - 備份檔案管理（保留天數）
- 手動備份 / 還原
  - 提供高權限管理員操作
  - 還原操作需雙重確認與 Log
- 系統健康檢查
  - 簡易顯示資料庫容量
  - 磁碟剩餘空間

## 十、技術與前端大致規格（概要）

> 此處先描述需求層級與模組區分，實際技術選型可在架構設計階段再定。

### 10.1 前端類型與需求

- **後台管理端**
  - 以 Web（桌機瀏覽器優化）為主
  - 需支援 RWD 至筆電 / 平板
  - 採登入授權管理

- **POS 收銀端**
  - 支援觸控螢幕操作（大按鈕、簡化操作步驟）
  - 條碼掃描器支援（以鍵盤模擬或 Serial / USB）
  - 若 Local Host 僅一台收銀機，可單機運作；若多台，則需與本地伺服器同步

### 10.2 後端與資料庫（需求層級）

- Local Host 架構
  - 資料庫部署於店內或公司伺服器
  - POS 與後台透過區域網路存取 API

- API 分層（僅列模組）
  - 基礎主檔 API
    - 商家 / 組織
    - 供應商
    - 商品
    - 倉庫 / 門市
    - 會員
  - 交易 API
    - 銷售
    - 退貨 / 換貨
    - 採購 / 進貨
    - 調撥
    - 盤點
  - CRM API
    - 點數與優惠券
    - 行銷活動
    - 分級與分群
  - 報表查詢 API
    - 營運報表
    - 庫存報表
    - 會員報表
    - 財務對帳報表

- 安全與權限
  - API Token 驗證
  - 依帳號角色限制 API 操作
